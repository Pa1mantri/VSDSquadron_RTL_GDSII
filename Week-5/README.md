## Preparing for Gate-Level Netlist Integration

The netlist to be used for GLS  ``6_final.v`` is generated  at the results folder after the physical implementation of the user_project_wrappper.

The ``6_final.v`` file was chosen because it is the ultimate post-placed, post-routed, and post-CTS (Clock Tree Synthesis) physical netlist generated by OpenROAD. Unlike the RTL file, this netlist accurately reflects the actual silicon implementation, containing all physical standard cell instantiations and their physically routed connections."

### Dependencies 
Since the 6_final.v generated verilog file contains all standard cells and their connections, we should also provide verilog models of these standard cells for simulation. These files are ```sky130_fd_sc_hd.v``` and ```primitives.v```
Here is the absolute path for these two files
```
/home/vsdsquadron/.volare/sky130A/libs.ref/sky130_fd_sc_hd/verilog/primitives.v
/home/vsdsquadron/.volare/sky130A/libs.ref/sky130_fd_sc_hd/verilog/sky130_fd_sc_hd
```

In ASIC design, a netlist is only "simulation-ready" if the simulator knows how to handle the physical additions that OpenROAD made to the file. Physical additions include clock buffers, delay buffers, Tie cell, Antenna diodes. All these cells verilog models are provided through ``sky130_fd_sc_hd.v`` file.
